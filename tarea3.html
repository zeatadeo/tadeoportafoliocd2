¿Qué es HTML5?

La base de todo en la Web

HTML (HyperText Markup Language) es el lenguaje de marcado que define la estructura y el contenido de toda página web. No es un lenguaje de programación — no tiene lógica, condiciones ni ciclos — es un lenguaje declarativo: le dices al navegador qué es cada cosa.

HTML5 es la quinta y actual versión del estándar, lanzada oficialmente en 2014 por el W3C. Trajo consigo etiquetas semánticas, soporte nativo para audio y video, APIs de JavaScript y mucho mejor soporte para dispositivos móviles.

💡

Analogía: Si una página web fuera una casa, HTML sería la estructura (paredes, puertas, ventanas), CSS sería la decoración (colores, muebles, estilo) y JavaScript sería la electricidad e instalaciones (lo que hace funcionar las cosas).

🏷️

🔗
Hipertexto

🌐
Universal
Funciona en todos los navegadores y dispositivos sin necesidad de instalar nada extra.

Estructura de un Documento

El esqueleto de toda página HTML5

Todo documento HTML5 tiene una estructura mínima obligatoria. Cada parte cumple una función específica que el navegador necesita para interpretar correctamente la página.

plantilla.html 
<!-- 1. Declaración de tipo de documento -->
<!DOCTYPE html>

<!-- 2. Raíz del documento -->
<html lang="es">

  <!-- 3. Cabecera: información para el navegador -->
  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <meta name="description" content="Mi primera página web"/>
    <title>Mi Primera Página</title>
    <link rel="stylesheet" href="estilos.css"/>
  </head>

  <!-- 4. Cuerpo: contenido visible -->
  <body>
    <h1>¡Hola, mundo!</h1>
    <p>Esta es mi primera página HTML5.</p>
    <script src="app.js"></script>
  </body>

</html>
⚠️

Importante: <!DOCTYPE html> no es una etiqueta HTML — es una instrucción al navegador para usar el estándar HTML5. Siempre debe ir en la primera línea.

<!DOCTYPE html>
Declara el tipo de documento como HTML5. Va antes de todo.
<html lang="es">
Elemento raíz. El atributo lang define el idioma para accesibilidad y SEO.
<head>
Contiene metadatos, título, enlaces CSS/JS. No se muestra en pantalla.
<meta charset>
Define la codificación de caracteres. UTF-8 soporta acentos, ñ y emojis.
<meta viewport>
Hace la página responsiva en móviles. Esencial para diseño adaptable.
<title>
Texto que aparece en la pestaña del navegador y en resultados de búsqueda.
<link>
Vincula recursos externos como hojas de estilos CSS o iconos.
<body>
Contiene todo el contenido visible de la página. Solo puede haber uno.
<script>
Incluye código JavaScript. Se recomienda colocarlo al final del body.

Etiquetas de Texto

Títulos, párrafos, listas y formato

Las etiquetas de texto son las más usadas en HTML. Organizan el contenido en títulos, párrafos, listas y permiten aplicar énfasis semántico al contenido, lo que es importante para la accesibilidad y el posicionamiento en buscadores.

texto.html 
<!-- TÍTULOS: h1 es el más importante, h6 el menor -->
<h1>Título principal (solo uno por página)</h1>
<h2>Subtítulo de sección</h2>
<h3>Subtítulo de subsección</h3>

<!-- PÁRRAFO -->
<p>Este es un párrafo normal. Contiene texto.
   Los saltos de línea en el código no afectan al resultado.</p>

<!-- ÉNFASIS SEMÁNTICO -->
<p>
  Texto <strong>muy importante (negrita)</strong> y
  texto <em>con énfasis (cursiva)</em> y
  texto <mark>resaltado</mark> y
  texto <del>tachado</del>.
</p>

<!-- LISTAS -->
<ul>  <!-- Lista sin orden (bullets) -->
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<ol>  <!-- Lista ordenada (numerada) -->
  <li>Aprender HTML</li>
  <li>Aprender CSS</li>
  <li>Crear proyectos</li>
</ol>

<!-- LÍNEA HORIZONTAL Y SALTO -->
<hr/>   <!-- Línea divisoria -->
<br/>   <!-- Salto de línea -->

<!-- CITAS -->
<blockquote>
  "La Web es para todos." — Tim Berners-Lee
</blockquote>

<!-- CÓDIGO -->
<p>Usa <code>console.log()</code> para depurar.</p>
<h1> – <h6>
Títulos jerárquicos. H1 es el principal, H6 el de menor importancia.
<p>
Párrafo de texto. El navegador agrega espacio vertical automáticamente.
<strong>
Texto importante. Se muestra en negrita y tiene significado semántico para lectores de pantalla.
<em>
Texto con énfasis. Se muestra en cursiva. No usar solo para decoración visual.
<ul> / <ol>
Lista sin orden (bullets) y lista ordenada (números). Contienen elementos <li>.
<li>
Elemento de lista. Siempre va dentro de <ul>, <ol> o <menu>.
<blockquote>
Cita larga de otra fuente. El atributo cite puede indicar la URL de origen.
<span>
Contenedor de texto en línea sin semántica propia. Útil para aplicar estilos CSS.
<mark>
Texto resaltado como con marcador amarillo. Útil para resultados de búsqueda.
<code> / <pre>
<code> para código en línea, <pre> para bloques de código preformateado.
<sub> / <sup>
Subíndice (H₂O) y superíndice (x²). Útiles en fórmulas matemáticas y notas.
<br/> / <hr/>
Salto de línea y línea horizontal divisoria. Son etiquetas vacías (sin cierre).

Enlaces e Hipervínculos

La esencia del hipertexto en la Web

Los enlaces son la característica que da nombre al lenguaje: HyperText. La etiqueta <a> (anchor) conecta páginas, secciones, archivos y recursos externos.

enlaces.html 
<!-- Enlace externo -->
<a href="https://www.google.com"
   target="_blank"
   rel="noopener noreferrer">
  Visitar Google
</a>

<!-- Enlace interno (misma carpeta) -->
<a href="contacto.html">Ir a Contacto</a>

<!-- Enlace a sección de la misma página (ancla) -->
<a href="#seccion-2">Ir a sección 2</a>
<section id="seccion-2">...</section>

<!-- Enlace de correo -->
<a href="mailto:tadeo@ejemplo.com">Enviar correo</a>

<!-- Enlace de teléfono (móviles) -->
<a href="tel:+529612345678">Llamar</a>

<!-- Enlace para descargar archivo -->
<a href="archivo.pdf" download>Descargar PDF</a>
AtributoValoresDescripción
hrefURL, #id, mailto:, tel:Destino del enlace. El más importante.
target_blank, _self, _parentDónde se abre: _blank = nueva pestaña, _self = misma pestaña.
relnoopener, noreferrerSeguridad: evita que la página destino acceda al origen con _blank.
download(sin valor o nombre)Fuerza la descarga del archivo en lugar de abrirlo.
titletextoTooltip que aparece al pasar el cursor sobre el enlace.

Imágenes y Multimedia

HTML5 nativo: sin Flash, sin plugins

HTML5 introdujo soporte nativo para imágenes responsivas, video y audio sin depender de plugins de terceros. La etiqueta <img> es la más usada, pero <video>, <audio> y <picture> son igualmente importantes.

multimedia.html 
<!-- IMAGEN básica -->
<img
  src="foto.jpg"
  alt="Descripción de la imagen"
  width="800"
  height="600"
  loading="lazy"
/>

<!-- IMAGEN responsiva con múltiples fuentes -->
<picture>
  <source media="(min-width:800px)" srcset="grande.jpg"/>
  <source media="(min-width:400px)" srcset="mediana.jpg"/>
  <img src="pequeña.jpg" alt="Mi imagen"/>
</picture>

<!-- VIDEO nativo HTML5 -->
<video
  src="video.mp4"
  controls
  autoplay
  muted
  loop
  poster="portada.jpg"
  width="100%"
>
  Tu navegador no soporta video HTML5.
</video>

<!-- AUDIO nativo -->
<audio controls src="cancion.mp3">
  Tu navegador no soporta audio HTML5.
</audio>

<!-- IFRAME (incrustar contenido externo) -->
<iframe
  src="https://www.youtube.com/embed/ID"
  width="560" height="315"
  frameborder="0"
  allowfullscreen
></iframe>
✅

Buena práctica: Siempre incluye el atributo alt en las imágenes. Es obligatorio para accesibilidad y mejora el SEO. Si la imagen es puramente decorativa, usa alt="".

<img>
Incrusta una imagen. Etiqueta vacía: no tiene cierre. src y alt son obligatorios.
<picture>
Contenedor para imágenes responsivas con múltiples fuentes según el dispositivo.
<video>
Reproductor de video nativo. Soporta MP4, WebM y Ogg sin plugins.
<audio>
Reproductor de audio. Soporta MP3, WAV y Ogg. El atributo controls muestra los controles.
<iframe>
Incrusta contenido externo como mapas de Google, videos de YouTube o páginas.
<canvas>
Área de dibujo programable con JavaScript. Base de juegos 2D y gráficas dinámicas.
<svg>
Gráficos vectoriales en línea. Se puede animar con CSS y manipular con JS.
<figure> / <figcaption>
Agrupa una imagen con su descripción. Mejora la semántica del documento.

Formularios

Cómo recibir datos del usuario

Los formularios son la principal forma de interacción entre el usuario y el servidor. HTML5 mejoró enormemente los formularios con nuevos tipos de input, validación nativa y mejor soporte para móviles.

formulario.html 
<form action="/enviar" method="POST">

  <!-- Texto -->
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre"
         placeholder="Tu nombre" required/>

  <!-- Email con validación automática -->
  <input type="email" name="correo"
         placeholder="correo@ejemplo.com"/>

  <!-- Número con rango -->
  <input type="number" name="edad"
         min="1" max="120"/>

  <!-- Selector de fecha -->
  <input type="date" name="fecha"/>

  <!-- Color picker -->
  <input type="color" name="color" value="#2563eb"/>

  <!-- Selector desplegable -->
  <select name="especialidad">
    <option value="ia">Inteligencia Artificial</option>
    <option value="web">Desarrollo Web</option>
  </select>

  <!-- Área de texto -->
  <textarea name="mensaje" rows="4"
    placeholder="Escribe tu mensaje..."></textarea>

  <!-- Checkbox y Radio -->
  <input type="checkbox" name="acepto" id="acepto"/>
  <label for="acepto">Acepto los términos</label>

  <!-- Botón de envío -->
  <button type="submit">Enviar formulario</button>

</form>
type="text"
Campo de texto de una sola línea. El tipo más común.
type="email"
Valida automáticamente el formato de correo. En móvil muestra teclado con @.
type="password"
Oculta el texto mientras se escribe. Esencial para contraseñas.
type="number"
Solo permite números. Permite definir min, max y step.
type="date"
Muestra un selector de fecha nativo del navegador.
type="file"
Permite subir archivos. Con multiple acepta varios a la vez.
type="checkbox"
Casilla de selección múltiple. El atributo checked la marca por defecto.
type="radio"
Selección única entre opciones con el mismo name.
type="range"
Control deslizante para seleccionar un valor en un rango.
required
Atributo que hace el campo obligatorio. El formulario no se envía si está vacío.
<label>
Etiqueta descriptiva de un campo. Su atributo for debe coincidir con el id del input.
<fieldset>
Agrupa campos relacionados. <legend> proporciona el título del grupo.

Tablas

Para datos tabulares, no para diseño

Las tablas HTML son para presentar datos tabulares — como una hoja de cálculo. Es un error común usarlas para diseño de layout; para eso existe CSS Grid y Flexbox.

tabla.html 
<table>
  <caption>Alumnos del grupo</caption>

  <!-- Cabecera -->
  <thead>
    <tr>
      <th scope="col">Nombre</th>
      <th scope="col">Materia</th>
      <th scope="col">Calificación</th>
    </tr>
  </thead>

  <!-- Cuerpo -->
  <tbody>
    <tr>
      <td>Brandon Tadeo</td>
      <td>Cultura Digital</td>
      <td>9.8</td>
    </tr>
    <tr>
      <td>Ana López</td>
      <td>Cultura Digital</td>
      <td>9.5</td>
    </tr>
  </tbody>

  <!-- Pie -->
  <tfoot>
    <tr>
      <td colspan="2">Promedio del grupo</td>
      <td>9.65</td>
    </tr>
  </tfoot>

</table>
<table>
Contenedor de la tabla. Todo debe ir dentro de esta etiqueta.
<thead> / <tbody> / <tfoot>
Dividen la tabla en cabecera, cuerpo y pie. Mejoran accesibilidad y estilo.
<tr>
Table Row — define una fila. Contiene celdas <th> o <td>.
<th>
Celda de encabezado. Se muestra en negrita. scope indica si es de columna o fila.
<td>
Table Data — celda de datos normal. colspan y rowspan permiten unir celdas.
<caption>
Título descriptivo de la tabla. Va justo después de <table>.

HTML5 Semántico

La evolución más importante de HTML5

La semántica es el uso de etiquetas con significado propio. Antes de HTML5, todo se hacía con <div> sin ningún significado descriptivo. HTML5 introdujo etiquetas que describen qué es el contenido, no solo cómo se ve.

El HTML semántico beneficia la accesibilidad (lectores de pantalla), el SEO (Google entiende mejor la página) y la mantenibilidad del código.

semantica.html 
<!-- Antes de HTML5 (sin semántica) -->
<div id="header">...</div>
<div id="nav">...</div>
<div id="main">...</div>
<div id="footer">...</div>

<!-- Con HTML5 semántico ✓ -->
<header>
  <nav>
    <ul>
      <li><a href="/">Inicio</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <header>
      <h1>Título del artículo</h1>
      <time datetime="2024-03-15">15 de marzo, 2024</time>
    </header>
    <section>
      <p>Contenido del artículo...</p>
    </section>
  </article>

  <aside>
    <h2>Artículos relacionados</h2>
  </aside>
</main>

<footer>
  <address>Contacto: tadeo@ejemplo.com</address>
</footer>
<header>
Cabecera de la página o de una sección. Contiene logo, título y navegación.
<nav>
Bloque de navegación principal. Menús, barras de navegación y breadcrumbs.
<main>
Contenido principal de la página. Solo puede haber uno y no debe estar anidado.
<article>
Contenido independiente y reutilizable: post de blog, noticia, comentario.
<section>
Sección temática de un documento. Siempre debe tener un encabezado.
<aside>
Contenido complementario al principal: sidebar, publicidad, links relacionados.
<footer>
Pie de página o sección. Contiene copyright, contacto y links secundarios.
<time>
Representa una fecha u hora. El atributo datetime da el valor legible para máquinas.
<details> / <summary>
Acordeón nativo HTML5. <summary> es el título visible; <details> el contenido colapsable.
<figure> / <figcaption>
Contenido ilustrativo (imagen, diagrama) con su leyenda descriptiva.
<address>
Información de contacto del autor o propietario del documento o artículo.
<div> / <span>
Sin semántica propia. Usar solo cuando ninguna etiqueta semántica sea apropiada.

"El HTML semántico no es una opción — es la diferencia entre una página que cualquiera puede entender y una que solo tú puedes mantener."

— Principio de desarrollo web moderno
